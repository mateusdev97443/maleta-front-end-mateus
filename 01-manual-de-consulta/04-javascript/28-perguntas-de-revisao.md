# Perguntas de revisão

Responda com suas palavras e, quando possível, crie um exemplo pequeno no console.

## Papel do JavaScript

1. Em uma página de produto, quais responsabilidades seriam do HTML, do CSS e do JavaScript?
2. Por que estudar lógica antes de conectar JavaScript à tela?
3. Cite três regras de negócio simples que JavaScript pode resolver antes de qualquer interface.

## Variáveis e tipos

4. Quando você escolheria `const` em vez de `let`?
5. Dê um exemplo real em que `let` é mais adequado que `const`.
6. Por que `var` não faz parte do padrão moderno da Maleta?
7. Qual a diferença prática entre o valor `22` e o valor `"22"`?
8. Como você confirmaria se uma variável é array?

## Comparações e decisões

9. Explique por que `"18" == 18` pode ser perigoso em código profissional.
10. Quando uma condição com `&&` deve ser usada?
11. Quando uma condição com `||` faz sentido?
12. Como você testaria uma regra de maioridade com valores de limite?
13. Por que a ordem de `if`, `else if` e `else` pode mudar o resultado?

## Listas e objetos

14. Como decidir entre array e objeto para representar um dado?
15. Quando faz sentido usar array de objetos?
16. O que acontece quando você acessa um índice que não existe?
17. Qual a diferença entre `push` e `map` quanto ao array original?
18. Em que situação `find` é melhor que `filter`?

## Funções

20. Por que funções de cálculo devem retornar valor?

23. Explique callback com suas palavras usando um exemplo simples.
