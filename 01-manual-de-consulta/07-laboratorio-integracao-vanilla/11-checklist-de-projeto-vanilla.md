# Checklist de projeto vanilla

## Checklist profissional

- [ ] Estados de loading, erro, vazio e sucesso implementados.
- [ ] Loading aparece no início e some no final.
- [ ] `response.ok` validado antes de `response.json()`.
- [ ] `response.status` usado quando ajuda a explicar falha.
- [ ] Lista limpa antes de nova renderização.
- [ ] Erro antigo limpo antes de nova tentativa.
- [ ] Dados originais preservados ao filtrar.
- [ ] Eventos registrados no lugar certo, de preferência na inicialização.
- [ ] Botão desabilitado quando uma ação repetida pode causar problema.
- [ ] Funções pequenas e com nomes claros.
- [ ] Sem duplicação desnecessária de render.
- [ ] Layout pensado para telas pequenas e grandes.
- [ ] Sem lógica de busca, regra e DOM misturada sem necessidade.
- [ ] Sem frameworks, bibliotecas externas, token exposto, backend ou GitHub Pages.
- [ ] Sem arquivos reais `.html`, `.css` ou `.js` criados para a fase.
