# 18 — Modelo de análise de HTML

Use este roteiro para auditar qualquer página.

## 1. Estrutura

- Existe documento HTML completo?
- O idioma está definido?
- `head` e `body` estão separados corretamente?
- Há `main` único e claro?

## 2. Semântica

- Tags estruturais representam o conteúdo real?
- `section` tem tema e título?
- `article` é usado para conteúdo independente?
- `div` aparece apenas quando não há tag melhor?

## 3. Acessibilidade

- Headings formam um sumário lógico?
- Imagens têm `alt` adequado?
- Formulários têm labels e grupos claros?
- Links e botões são compreensíveis?

## 4. SEO

- `title` é único e específico?
- `description` é útil?
- O `h1` comunica o tema principal?
- Links internos ajudam a entender a estrutura?

## 5. Formulários

- Campos têm `name`, `id` e `label`?
- Tipos de input estão corretos?
- Validações HTML ajudam sem atrapalhar?

## 6. Imagens e mídia

- `alt`, `width` e `height` foram considerados?
- Iframes têm `title`?
- Mídias têm controles quando necessário?

## 7. Erros e melhorias

Registre problemas em três níveis: crítico, importante e melhoria futura. Termine com ações objetivas, por exemplo: “conectar label ao input de e-mail” em vez de “melhorar formulário”.
