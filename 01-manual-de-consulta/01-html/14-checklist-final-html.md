# 14 — Checklist final de HTML

Use este checklist antes de considerar uma página pronta.

## Documento

- [ ] Existe `<!doctype html>`?
- [ ] `html` tem `lang` correto?
- [ ] `charset`, `viewport`, `title` e `description` estão presentes?
- [ ] O `title` é específico da página?

## Estrutura

- [ ] Existe um `main` claro?
- [ ] `header`, `nav`, `section`, `article`, `aside` e `footer` foram usados quando fazem sentido?
- [ ] A página continua compreensível sem CSS?

## Conteúdo

- [ ] Existe apenas um assunto principal evidente?
- [ ] Headings seguem ordem lógica?
- [ ] Parágrafos e listas estão bem divididos?

## Acessibilidade

- [ ] Imagens informativas têm `alt` adequado?
- [ ] Formulários têm labels conectados?
- [ ] Links e botões têm texto claro?
- [ ] A navegação por teclado segue ordem natural?

## Qualidade

- [ ] Não há `div` desnecessária para tudo?
- [ ] Não há tabela usada para layout?
- [ ] Não há excesso de `br` para espaçamento?
- [ ] O HTML foi validado ou revisado estruturalmente?
