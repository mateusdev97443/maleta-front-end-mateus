# 11 — Erros comuns em HTML

## Lista de problemas frequentes

- Usar várias tags `h1` sem necessidade clara.
- Usar `div` para tudo e perder semântica.
- Inserir imagem informativa sem `alt`.
- Criar botão com link quando não há navegação.
- Usar link como botão sem tratar semântica e acessibilidade.
- Deixar `label` desconectado do `input`.
- Usar `table` para layout.
- Esquecer `meta viewport`.
- Usar `title` genérico.
- Criar estrutura sem `main`.
- Criar `section` sem título ou tema.
- Usar excesso de `br` para espaçamento.

## Como corrigir

- Troque agrupamentos genéricos por tags semânticas quando houver significado.
- Use CSS para espaçamento, não quebras de linha artificiais.
- Conecte `label for` ao `input id`.
- Use `button` para ações e `a` para navegação.
- Dê nomes específicos para títulos, seções e links.
