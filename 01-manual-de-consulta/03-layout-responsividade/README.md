# Manual Layout e Responsividade

## Objetivo da fase

A FASE 4 organiza o estudo de layout e responsividade da Maleta de Ferramentas Front-end. O objetivo é ensinar como transformar HTML semântico e CSS bem escrito em páginas adaptáveis, legíveis e profissionais, sem depender de JavaScript.

## Quando usar este manual

Use este manual quando precisar criar estrutura de página, organizar seções, alinhar elementos, montar cards, definir containers, aplicar Flexbox, aplicar Grid, planejar breakpoints ou revisar se um layout funciona bem em telas pequenas, médias e grandes.

## Pré-requisitos

Antes de iniciar, revise HTML e CSS básico/intermediário das fases anteriores. Esta fase assume que você já conhece tags semânticas, seletores, box model, cores, tipografia, unidades, classes e organização básica de arquivos.

## Como estudar a fase

Leia os capítulos na ordem, reproduza os códigos, altere valores e teste cada exemplo em larguras diferentes. Não memorize propriedades isoladas: observe qual problema cada técnica resolve.

## Ordem recomendada de estudo

Comece pelos capítulos 1 a 4 para dominar o comportamento natural dos elementos. Depois avance para Flexbox, Grid, espaçamentos, media queries e mobile-first. Em seguida estude imagens, cards, menus estruturais e layouts práticos. Finalize com snippets, checklist, exercícios, desafios, revisão, modelo de análise e padrão profissional.

## Avisos importantes

Esta fase não usa JavaScript. Menus responsivos aparecem apenas como conceito estrutural com HTML e CSS, sem interação dinâmica, botão hambúrguer funcional ou manipulação de estado.

## Conexão com a Fábrica de Projetos Front-end

A Fábrica de Projetos Front-end deve usar este manual como referência para montar páginas reais: landing pages, formulários, dashboards simples, cards, galerias e seções reutilizáveis. O padrão desta fase serve como base para projetos posteriores.

## Critérios para considerar a fase concluída

A fase pode ser considerada concluída quando todos os capítulos forem lidos, os exemplos principais forem reproduzidos, os exercícios graduais forem praticados, o checklist for aplicado em pelo menos uma página e o aluno conseguir justificar por que escolheu Flexbox, Grid, container, breakpoint e estratégia mobile-first em um layout.
