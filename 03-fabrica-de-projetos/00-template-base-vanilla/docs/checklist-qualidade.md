# Checklist de Qualidade

## HTML

- [ ] A página usa landmarks semânticos: `header`, `nav`, `main`, `section` e `footer`.
- [ ] Existe apenas um `h1` claro.
- [ ] Links possuem texto compreensível fora de contexto.

## CSS

- [ ] O reset fica separado de estilos visuais.
- [ ] Variáveis globais ficam em `variables.css`.
- [ ] O layout foi pensado primeiro para telas pequenas.
- [ ] Componentes reutilizáveis ficam em `components.css`.

## JavaScript

- [ ] O script usa `defer` ou é carregado no final do documento.
- [ ] Seletores inexistentes são tratados antes de manipular elementos.
- [ ] Não há dependências externas.

## Acessibilidade

- [ ] O foco visível aparece em elementos interativos.
- [ ] A navegação principal possui nome acessível.
- [ ] A leitura do conteúdo faz sentido sem CSS.

## Entrega

- [ ] README atualizado.
- [ ] Briefing preenchido.
- [ ] Projeto revisado em celular e desktop.
- [ ] Sem arquivos temporários ou conteúdo inútil.
