# Landing Pages

## Objetivo da categoria

Criar páginas únicas com foco em conversão: apresentar uma oferta, explicar seu valor e conduzir a pessoa visitante para uma ação principal, como comprar, cadastrar-se, solicitar contato ou baixar um material.

Uma boa landing page deve ser clara, rápida de entender, visualmente consistente e objetiva. O projeto treina hierarquia visual, escrita de interface, responsividade, acessibilidade e organização de seções em uma página comercial.

## Para quem é esse projeto

- Estudantes que já conhecem HTML e CSS básicos.
- Pessoas que querem montar páginas de produto, serviço, evento ou captura de leads.
- Iniciantes que precisam praticar layout responsivo com uma entrega realista.
- Portfólios que precisam mostrar capacidade de transformar briefing em interface.

## O que construir

Construa uma landing page completa para uma oferta fictícia ou real. Escolha um tema simples, como curso online, aplicativo, consultoria, evento, produto digital, restaurante, academia ou ferramenta de produtividade.

A página deve ter uma ação principal bem definida. Exemplos:

- “Começar agora”.
- “Solicitar orçamento”.
- “Reservar vaga”.
- “Baixar material”.
- “Entrar na lista de espera”.

## Seções obrigatórias

- **Cabeçalho:** marca, navegação curta e botão de ação.
- **Hero:** título forte, subtítulo, chamada para ação e imagem ou elemento visual.
- **Problema ou contexto:** explique a dor que a oferta resolve.
- **Benefícios:** liste de 3 a 6 ganhos claros para a pessoa usuária.
- **Como funciona:** mostre etapas simples do processo.
- **Prova de confiança:** depoimentos, números, selos, clientes ou garantia fictícia sinalizada como exemplo.
- **Oferta ou planos:** apresente o que está incluído.
- **Perguntas frequentes:** responda dúvidas comuns.
- **Chamada final:** repita a ação principal.
- **Rodapé:** informações básicas, links úteis e direitos autorais.

## Componentes sugeridos

- Botões primários e secundários.
- Cards de benefício.
- Lista de etapas numeradas.
- Bloco de depoimento.
- Tabela simples de planos ou comparação.
- Accordion de perguntas frequentes, se usar JavaScript.
- Banner ou card de destaque.
- Ícones decorativos com texto alternativo adequado quando necessário.

## Requisitos de HTML

- Use estrutura semântica com `header`, `main`, `section`, `article`, `nav` e `footer` quando fizer sentido.
- Tenha apenas um `h1` principal.
- Organize os subtítulos em ordem lógica, sem pular níveis de heading sem motivo.
- Use links reais para navegação interna por âncoras.
- Associe botões e links à ação correta: link navega, botão executa comportamento.
- Inclua textos alternativos em imagens informativas.
- Evite criar layout com tabelas.

## Requisitos de CSS e responsividade

- Desenvolva primeiro para telas pequenas e depois adapte para telas maiores.
- Use unidades flexíveis como `%`, `rem`, `fr`, `minmax()` e `clamp()` quando adequado.
- Garanta bom espaçamento entre seções.
- Mantenha contraste suficiente entre texto e fundo.
- Use `flexbox` e/ou `grid` para organizar cards e colunas.
- Evite largura fixa que quebre em celulares.
- Teste pelo menos três faixas: celular, tablet e desktop.

## Requisitos de JavaScript

JavaScript é opcional em landing pages simples. Use apenas se melhorar a experiência.

Boas possibilidades:

- Menu mobile acessível.
- Accordion de FAQ.
- Validação simples de formulário de captura.
- Scroll suave respeitando preferências de movimento.
- Contador visual ou interação leve, sem bloquear o conteúdo.

Evite depender de JavaScript para exibir o conteúdo principal da página.

## Acessibilidade mínima

- Todos os elementos interativos devem ser acessíveis por teclado.
- O foco visível não deve ser removido.
- Botões e links precisam ter nomes compreensíveis.
- Imagens decorativas devem usar `alt=""`.
- Formulários devem ter `label` associado ao campo.
- A página deve continuar compreensível sem animações.

## Critérios de aceite

- A proposta da landing page é entendida nos primeiros segundos.
- Existe uma ação principal clara e repetida em pontos estratégicos.
- Todas as seções obrigatórias foram implementadas.
- O layout não quebra em telas pequenas.
- A navegação interna funciona.
- O HTML passa por uma revisão semântica básica.
- A página tem aparência finalizada, sem textos genéricos como “lorem ipsum”.

## Variações de dificuldade

- **Iniciante:** landing page estática com hero, benefícios, FAQ e CTA final.
- **Intermediário:** menu mobile, formulário validado e seção de planos responsiva.
- **Avançado:** múltiplos blocos de prova social, animações discretas, métricas e testes de acessibilidade mais rigorosos.

## Erros comuns

- Começar pelo visual sem definir a ação principal.
- Usar muitos CTAs diferentes na mesma página.
- Escrever títulos vagos que não explicam a oferta.
- Deixar a versão mobile como adaptação de última hora.
- Usar contraste baixo em botões e textos.
- Inserir imagens pesadas sem necessidade.

## Checklist final

- [ ] A oferta está clara no hero.
- [ ] O CTA principal aparece no início e no fim da página.
- [ ] As seções seguem uma ordem lógica de convencimento.
- [ ] Todos os links e botões foram testados.
- [ ] A página foi revisada em celular e desktop.
- [ ] Não há textos de rascunho.
- [ ] O foco do teclado está visível.
- [ ] Imagens possuem `alt` adequado.

## Próximos passos

Depois de concluir a primeira versão, peça feedback sobre clareza, revise o texto dos CTAs, teste a página em dispositivos diferentes e documente no portfólio qual problema a landing page resolve e quais decisões de layout foram tomadas.
