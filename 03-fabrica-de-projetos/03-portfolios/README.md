# Portfólios

## Objetivo da categoria

Criar portfólios profissionais para apresentar habilidades, projetos, trajetória e formas de contato. O objetivo não é apenas mostrar telas bonitas, mas explicar problemas resolvidos, decisões tomadas e resultados alcançados.

Esse projeto treina curadoria de conteúdo, narrativa profissional, organização visual, acessibilidade e apresentação clara de projetos.

## Para quem é esse projeto

- Estudantes que querem publicar seus primeiros projetos.
- Pessoas em transição de carreira que precisam organizar experiências.
- Desenvolvedores iniciantes que querem demonstrar fundamentos de front-end.
- Profissionais que precisam de uma página pessoal simples e objetiva.

## O que construir

Construa um portfólio pessoal com informações reais ou fictícias. Ele deve apresentar quem é a pessoa, quais habilidades possui, quais projetos já desenvolveu e como entrar em contato.

Cada projeto exibido deve ter contexto, tecnologias usadas e um link para código, demonstração ou estudo de caso quando existir.

## Seções ou páginas obrigatórias

- **Apresentação:** nome, área de atuação, proposta profissional e chamada para contato.
- **Sobre:** breve resumo da trajetória, interesses e forma de trabalho.
- **Habilidades:** tecnologias, ferramentas e competências organizadas por categoria.
- **Projetos:** cards ou estudos de caso com descrição clara.
- **Experiência ou formação:** cursos, experiências, certificações ou aprendizados relevantes.
- **Contato:** e-mail, redes profissionais e chamada para conversa.
- **Rodapé:** links principais e créditos quando necessário.

## Componentes sugeridos

- Hero pessoal com CTA.
- Cards de projeto.
- Tags de tecnologia.
- Timeline de estudos ou experiência.
- Lista de habilidades por categoria.
- Botões para GitHub, LinkedIn, currículo e e-mail.
- Modal ou página de detalhes do projeto em versões avançadas.
- Alternância de tema claro/escuro, se fizer sentido.

## Requisitos de HTML

- Use `main` para o conteúdo principal e seções bem nomeadas.
- Mantenha apenas um `h1` com o nome ou posicionamento principal.
- Use headings claros para cada seção.
- Em cards de projeto, use título, descrição, lista de tecnologias e links explícitos.
- Não transforme todo o card em link se isso prejudicar acessibilidade; ofereça links identificáveis.
- Use `alt` adequado em capturas de tela ou imagens de projeto.
- Evite esconder informações importantes apenas em efeitos de hover.

## Requisitos de CSS e responsividade

- Crie identidade visual consistente, mas priorize legibilidade.
- Organize projetos em uma coluna no mobile e em grid em telas maiores.
- Use espaçamento suficiente entre seções para facilitar leitura.
- Garanta que botões e links tenham área de clique confortável.
- Evite animações excessivas que atrapalhem o conteúdo.
- Teste textos longos em cards para evitar quebras visuais.
- Mantenha contraste adequado em temas claros ou escuros.

## Requisitos de JavaScript

JavaScript é opcional. Use quando houver benefício real para a apresentação.

Possibilidades úteis:

- Filtro de projetos por tecnologia.
- Menu mobile.
- Alternância de tema com preferência salva em `localStorage`.
- Expansão de detalhes de projeto.
- Validação de formulário de contato, se houver formulário.

Não dependa de JavaScript para que links de projeto e contato funcionem.

## Acessibilidade mínima

- Links externos devem ter texto claro sobre o destino.
- O foco visível deve funcionar em toda a navegação.
- Informações não devem depender apenas de cor.
- Ícones de redes sociais precisam de nome acessível.
- Animações devem ser discretas e respeitar `prefers-reduced-motion` quando usadas.
- O portfólio deve ser navegável por teclado.

## Critérios de aceite

- A pessoa visitante entende rapidamente quem você é e o que você faz.
- Há pelo menos três projetos ou estudos apresentados com contexto.
- Cada projeto informa objetivo, tecnologias e aprendizado ou resultado.
- Os links de contato e redes funcionam.
- O layout está responsivo e legível.
- O conteúdo parece final, sem placeholders.
- A apresentação valoriza clareza profissional, não apenas estética.

## Variações de dificuldade

- **Iniciante:** página única com sobre, habilidades, projetos e contato.
- **Intermediário:** filtros de projeto, currículo para download e tema claro/escuro.
- **Avançado:** estudos de caso detalhados, páginas individuais de projeto, métricas e melhorias de performance.

## Erros comuns

- Listar tecnologias sem mostrar projetos que as usam.
- Colocar muitos efeitos visuais e pouca explicação.
- Usar descrições vagas como “projeto feito em HTML e CSS”.
- Não incluir links para contato.
- Exibir projetos quebrados ou incompletos como se estivessem finalizados.
- Copiar textos genéricos que não refletem a trajetória da pessoa.

## Checklist final

- [ ] O hero explica nome, área e proposta profissional.
- [ ] A seção de projetos tem descrições úteis.
- [ ] Links para código, demo ou estudo de caso foram testados.
- [ ] Habilidades estão organizadas por contexto.
- [ ] Contato está fácil de encontrar.
- [ ] O layout foi revisado no mobile.
- [ ] Ícones e imagens têm nomes ou textos alternativos adequados.
- [ ] Não há conteúdo provisório.

## Próximos passos

Depois de publicar a primeira versão, escolha os melhores projetos para destacar, escreva estudos de caso curtos, peça revisão de clareza para outra pessoa e atualize o portfólio sempre que concluir um projeto relevante.
