Fluxo de uso

Como usar a Maleta

Use a Maleta como uma rotina de estudo e entrega: consulte, pratique, revise, registre e só então avance.

  1. Consultar: revise os manuais antes de implementar conceitos.
  2. Praticar: use exercícios, snippets e projetos vanilla com escopo claro.
  3. Validar: aplique checklists de qualidade antes de considerar uma entrega pronta.
  4. Registrar: documente decisões e aprendizados no dossiê de evolução.

Padrão de qualidade

Critérios antes de avançar

A Maleta prioriza fundamentos sólidos, escopo explícito e revisão cuidadosa antes de adicionar complexidade.

  • HTML semântico, `lang="pt-BR"`, viewport e headings coerentes.
  • CSS mobile-first, responsivo, com contraste adequado e foco visível.
  • JavaScript vanilla com funções claras, estados de interface e DOM seguro.
  • Sem frameworks, bibliotecas externas, npm ou build obrigatório nesta etapa.
  • Documentação alinhada ao que realmente foi implementado.
