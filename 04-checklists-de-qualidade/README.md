# Checklists de Qualidade

## Objetivo

Centralizar listas de verificação para revisar páginas, projetos, documentação e organização antes de considerar uma entrega pronta.

## Quando usar

Use esta área antes de concluir uma fase, publicar uma página, abrir uma revisão, avançar para novo projeto ou consolidar mudanças estruturais na Maleta.

## Como organizar

- Cada checklist deve ter escopo explícito.
- Itens devem ser verificáveis, não genéricos.
- Separar critérios de HTML, CSS, JavaScript, acessibilidade, UX, segurança, performance e documentação quando fizer sentido.
- Manter linguagem objetiva e orientada à ação.

## Status atual

**Em evolução.** A área começou como suporte planejado e agora passa a concentrar validações globais da Maleta.

## O que já existe

- README da área.
- Checklist de auditoria do repositório para validação local e revisão pós-auditoria.

## O que será evoluído depois

- Checklist para páginas HTML estáticas.
- Checklist para projetos JavaScript vanilla.
- Checklist de acessibilidade prática.
- Checklist de documentação e escopo.

## Regra para não poluir a Maleta

Não criar checklist duplicado para cada pequena alteração. Prefira checklists reutilizáveis e atualize os existentes quando o critério for geral.
