# Maleta Front-end Mateus

Repositório profissional de consulta, prática e evolução para o estudante de Front-end Mateus.

## Objetivo

Centralizar manuais, snippets, checklists, prompts e uma fábrica de projetos para acelerar o aprendizado com organização, consistência e foco em qualidade.

## Como usar esta maleta

1. Consulte o `01-manual-de-consulta/` antes de implementar uma tecnologia ou revisar conceitos.
2. Reaproveite padrões de `02-snippets-e-componentes/` somente depois de entender o que eles fazem.
3. Crie projetos a partir de `03-fabrica-de-projetos/`, começando pelo template base vanilla.
4. Valide entregas com `04-checklists-de-qualidade/` antes de considerar um projeto finalizado.
5. Use `05-prompts-codex/` para pedir revisão, explicação, refatoração e planejamento com mais precisão.
6. Registre aprendizados e evolução em `06-dossie-de-evolucao/`.
7. Mova materiais obsoletos para `99-arquivo/`, mantendo histórico sem poluir a base principal.

## Estrutura

```text
maleta-front-end-mateus/
├── 00-sistema-operacional/      # Regras de uso, rotina, padrões e organização da maleta
├── 01-manual-de-consulta/       # Guias técnicos por assunto
├── 02-snippets-e-componentes/   # Trechos reutilizáveis e componentes simples
├── 03-fabrica-de-projetos/      # Templates e categorias de projetos práticos
├── 04-checklists-de-qualidade/  # Critérios de revisão antes de entregar
├── 05-prompts-codex/            # Prompts para estudo, revisão e criação de projetos
├── 06-dossie-de-evolucao/       # Registro de progresso, decisões e retrospectivas
└── 99-arquivo/                  # Materiais antigos, substituídos ou pausados
```

## Princípios da maleta

- **Clareza antes de volume:** cada arquivo deve resolver uma necessidade real.
- **Base vanilla primeiro:** HTML, CSS e JavaScript sem frameworks até consolidar fundamentos.
- **Mobile-first:** todo projeto nasce funcionando bem em telas pequenas.
- **Acessibilidade como padrão:** semântica, contraste, foco visível e navegação por teclado.
- **Evolução registrada:** aprendizados e decisões importantes devem ficar documentados.
