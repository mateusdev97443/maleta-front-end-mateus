# Prompts Codex

## Objetivo

Organizar prompts para estudar, revisar código, planejar projetos, auditar entregas e pedir refatorações com mais precisão.

## Quando usar

Use esta área quando precisar de um pedido estruturado para revisão técnica, explicação de conceito, planejamento de fase, auditoria de acessibilidade, análise de bug ou melhoria de documentação.

## Como organizar

- Cada prompt deve ter objetivo claro.
- Informar contexto, escopo, restrições e resultado esperado.
- Evitar prompts genéricos que não orientam a revisão.
- Separar prompts por tipo de uso quando a pasta crescer.

## Status atual

**Em evolução.** A área é suporte planejado para melhorar a qualidade das interações com IA sem substituir revisão humana.

## O que já existe

- README com objetivo e regras iniciais.
- Espaço reservado para prompts validados em fases futuras.

## O que será evoluído depois

- Prompts de auditoria técnica.
- Prompts de explicação didática.
- Prompts de revisão de PR.
- Prompts de planejamento de projetos vanilla.

## Regra para não poluir a Maleta

Não guardar prompts longos sem uso comprovado. Um prompt só deve permanecer aqui se melhorar clareza, revisão ou aprendizado.
