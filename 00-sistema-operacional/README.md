# Sistema Operacional da Maleta

## Objetivo

Definir regras, rotinas e padrões para manter a Maleta útil, limpa, escalável e coerente com a evolução Front-end vanilla.

## Quando usar

Use esta área antes de reorganizar pastas, criar novas fases, mover arquivos, revisar status ou decidir se um conteúdo deve entrar na base ativa, no backlog ou no arquivo.

## Como organizar

- Registrar regras de uso e manutenção da Maleta.
- Documentar decisões que afetam a estrutura geral.
- Manter nomes claros e previsíveis.
- Evitar arquivos duplicados ou sem contexto.
- Atualizar índices quando a estrutura crescer.

## Status atual

**Em evolução.** A área existe como suporte de governança da Maleta. Ela não invalida as fases concluídas; serve para amadurecer a organização antes de novas fases.

## O que já existe

- README com objetivo, modo de uso e regras iniciais de organização.
- Direção clara para manter a Maleta como base viva, não como coleção solta de arquivos.

## O que será evoluído depois

- Rotina de revisão antes de novas fases.
- Padrões para nomear arquivos e categorias.
- Critérios para mover materiais para `99-arquivo/`.
- Modelo de registro de decisões estruturais.

## Regra para não poluir a Maleta

Só adicione regras que ajudem manutenção real. Evite transformar esta pasta em lista genérica de boas intenções ou duplicar conteúdo que pertence aos manuais técnicos.
