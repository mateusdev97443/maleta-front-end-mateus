# Snippets e Componentes

## Objetivo

Reunir pequenos padrões reutilizáveis de HTML, CSS e JavaScript vanilla para apoiar estudo e prática sem substituir entendimento dos fundamentos.

## Quando usar

Use esta área quando precisar consultar um padrão pequeno, isolado e reutilizável, como estrutura de card, botão acessível, formulário simples, estado visual ou interação básica com DOM.

## Como organizar

- Um snippet deve ter objetivo claro.
- O código deve ser pequeno e legível.
- O exemplo deve indicar quando usar e quando evitar.
- Snippets não devem depender de frameworks, bibliotecas ou build.
- Não adicionar componente sem contexto didático.

## Status atual

**Em evolução.** A área é suporte planejado da Maleta e será expandida conforme os projetos vanilla exigirem padrões reutilizáveis.

## O que já existe

## O que será evoluído depois

- Exemplos alinhados aos projetos da Fábrica de Projetos.

## Regra para não poluir a Maleta

Não salvar código copiado sem revisão. Um snippet só deve entrar aqui se for reutilizável, compreensível e alinhado com os princípios da Maleta.
